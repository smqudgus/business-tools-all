# business-tools-all-auto-price-monitor-v2

수정 내용:
- 배송비 3,000원 같은 잘못된 가격 제외
- 입고가의 50% 미만, 우리 판매가의 3배 초과 가격은 실패 처리
- 쿠팡/네이버 자동수집 실패 시 수동 가격 보정 가능
- 수동 입력도 price_history에 저장

- v2.1: 상품명 검색 input 디자인 정렬 수정


## v3 수정 내용
- 기존 DB에 이미 저장된 잘못된 가격(예: 메가커피 3,000원)을 화면 최신가에서 제외합니다.
- `api/products.js`에서 입고가의 50% 미만 / 우리 판매가의 3배 초과 가격을 표시하지 않도록 필터링합니다.
- `cleanup_wrong_price_history.sql`을 추가했습니다. Supabase SQL Editor에서 실행하면 기존 잘못된 기록을 삭제합니다.


## v4 수정 내용
- 메가커피 50,000원처럼 무료배송 기준금액/혜택금액으로 추정되는 숫자가 가격으로 잡히는 문제를 줄였습니다.
- 정상 가격 허용 범위를 `입고가의 80% 이상 ~ 우리 판매가의 1.5배 이하`로 강화했습니다.
- 기존에 저장된 50,000원 기록은 `cleanup_wrong_price_history.sql` 실행으로 삭제하세요.


## v5 수정 내용
- 메가커피에서 50,000원이 다시 생성되는 문제 방지.
- 정상 가격 허용 범위를 `입고가의 95% 이상 ~ 우리 판매가의 1.5배 이하`로 더 강화했습니다.
- 로투스 입고가 71,720원 기준 68,134원 미만은 자동수집 가격으로 인정하지 않습니다.
- `cleanup_wrong_price_history.sql` 실행 시 기존 50,000원 기록도 삭제합니다.


## v6 수정 내용
- 메가커피 페이지의 `총 합계 금액` 근처 가격을 최우선으로 수집합니다.
- 총 합계 금액을 못 찾으면 정상 범위 후보 중 우리 판매가와 가장 가까운 금액을 선택합니다.
- 요거라이트처럼 옵션 단가 8,950원이 대표 상품가로 잡히는 문제를 줄였습니다.
- 기존 잘못된 기록 제거용 `cleanup_wrong_price_history.sql`을 포함했습니다.


## v7 수정 내용
- 메가커피뿐 아니라 빅커피/쿠팡/네이버 URL에도 동일한 가격 선택 기준을 적용합니다.
- 1순위: 총 합계 금액 / 합계 금액 / 결제 금액 주변 가격
- 2순위: 정상 범위 안의 가격 중 우리 판매가와 가장 가까운 금액
- 3순위: 자동 수집 실패 시 수동 가격 보정
- 쿠팡/네이버는 서버에서 받은 HTML에 가격이 없을 수 있으므로 실패 가능성이 있습니다.


## v8 수정 내용
- 상품 상세 영역에 `URL/가격 수정` 버튼을 추가했습니다.
- 선택한 상품의 상품명/입고가/우리 판매가/메가커피/빅커피/쿠팡/네이버 URL을 아래 수정 폼으로 불러옵니다.
- 수정 후 `상품 저장`을 누르면 기존 상품 정보가 업데이트됩니다.


## v9 수정 내용
- 네이버 스마트스토어 HTML에서 `kakao:commerce:price`, `kakao:commerce:regular_price`를 우선 읽습니다.
- 네이버 초기 데이터의 `discountedSalePrice`, `salePrice`, `totalPaymentAmount`, `totalOrderAmount`도 가격 후보로 읽습니다.
- 네이버 URL이 일반 상품 페이지여도 대표 가격을 자동수집할 가능성을 높였습니다.


## v10 수정 내용
- 상품 상세 영역에 `URL/가격 수정` 버튼을 강제로 추가했습니다.
- 선택 상품의 기존 상품명/입고가/우리 판매가/메가커피/빅커피/쿠팡/네이버 URL을 수정 폼으로 불러옵니다.
- 수정 후 `상품 저장`을 누르면 기존 상품 정보가 업데이트됩니다.


## v11 수정 내용
- 네이버 가격은 `kakao:commerce:price`, `salePrice` 등 신뢰 가능한 데이터가 있으면 입고가 95% 필터를 적용하지 않습니다.
- 네이버 최저가는 우리 입고가보다 낮을 수 있으므로 기존 필터 때문에 `-`가 뜨던 문제를 수정했습니다.
- 메가커피/빅커피 일반 HTML 수집은 기존 오수집 방지 필터를 유지합니다.


## v12 수정 내용
- 쿠팡/네이버처럼 수집이 실패할 때 단순 `-` 대신 실패 원인을 확인할 수 있도록 보강했습니다.
- 실패 기록은 `price_history.note`에 저장됩니다. 예: `HTTP 429`, `HTTP 403`
- 진단 API를 추가했습니다.
  `/api/debug-check?url=상품URL&cost=입고가&our_price=판매가`
- HTTP 403은 접근 거부, HTTP 429는 요청 제한/봇 차단 가능성이 큽니다.


## v13 수정 내용
- 현재 가격 비교표에서 최신 수집이 실패한 판매처를 `-`가 아니라 `수집실패(429)`, `수집실패(403)`으로 표시합니다.
- `api/products`가 판매처별 최신 실패 기록까지 내려주도록 수정했습니다.
- 즉시 가격 확인 후 상단 상태문구에도 일부 실패 사유를 같이 표시합니다.
